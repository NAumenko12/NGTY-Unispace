# NGTY UniSpace

Fullstack-платформа для бронирования университетских коворкинг-пространств. Проект объединяет интерактивную карту корпусов, бронирование рабочих мест, личный кабинет, электронную библиотеку, чат и административную панель.

## Возможности

- регистрация и авторизация пользователей;
- роли `student` и `admin`;
- бронирование рабочих мест по корпусу, дате и времени;
- проверка пересечений бронирований;
- автоматическое обновление статусов бронирований;
- личный кабинет с историей бронирований;
- интерактивная карта корпусов;
- электронная библиотека учебных материалов;
- чат между пользователями;
- загрузка вложений в чат;
- админ-панель со статистикой, пользователями, бронированиями и библиотекой;
- бан и разбан пользователей;
- security headers через middleware.

## Backend

Backend реализован через Next.js Route Handlers в папке `app/api`.

Основные backend-модули:

- `auth` — регистрация, вход, выход и текущая сессия;
- `bookings` — создание, получение и отмена бронирований;
- `buildings` — корпуса университета;
- `workspaces` — рабочие места и зоны;
- `library` — электронная библиотека;
- `chat` — сообщения, контакты и загрузка вложений;
- `admin` — статистика, пользователи, бронирования, библиотека, бан/разбан.

## База данных

Проект использует SQLite. Схема создается скриптом `scripts/init-db.js`.

Основные таблицы:

- `users` — пользователи, роли, профиль, бан;
- `buildings` — корпуса университета с координатами;
- `workspaces` — рабочие места внутри корпусов;
- `bookings` — бронирования рабочих мест;
- `messages` — сообщения пользователей;
- `chat_deleted_conversations` — скрытые диалоги;
- `library_materials` — материалы электронной библиотеки.

Также создаются индексы для ускорения запросов по бронированиям, сообщениям, категориям и предметам библиотеки.

## Авторизация и безопасность

- пароли хешируются через `bcryptjs`;
- сессия хранится в cookie;
- payload сессии шифруется через `jose` / JWE `A256GCM`;
- доступ к защищенным API проверяется через `requireAuth`;
- админские API защищены через `requireAdmin`;
- заблокированные пользователи не получают активную сессию;
- middleware добавляет security headers:
  - `Content-Security-Policy`;
  - `X-Frame-Options`;
  - `X-Content-Type-Options`;
  - `Referrer-Policy`;
  - `Permissions-Policy`.

## Бронирования

Backend проверяет:

- обязательные параметры бронирования;
- что время не находится в прошлом;
- что рабочее место свободно в выбранном интервале;
- пересечения по времени с активными бронированиями;
- возможность отмены бронирования.

Статусы бронирований:

- `active`;
- `in_progress`;
- `completed`;
- `cancelled`;
- `expired`.

## Чат

В чате реализовано:

- получение контактов;
- отправка сообщений;
- отметка сообщений как прочитанных;
- скрытие удаленных диалогов для конкретного пользователя;
- загрузка вложений до 10 МБ.

## Запуск проекта

Установить зависимости:

```bash
npm install
```

Создать `.env.local`:

```bash
cp .env.example .env.local
```

Инициализировать базу данных:

```bash
npm run init-db
```

Запустить dev-сервер:

```bash
npm run dev
```

Приложение будет доступно по адресу:

```text
http://localhost:3000
```
